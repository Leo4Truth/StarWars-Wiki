# 《星球大战》多语种素材与背景知识采集方案

## 1. 目标

构建可持续更新的知识底座，覆盖电影、剧集、小说、漫画等内容，并支持简体中文与英文双语输出，后续可扩展其他语言。

## 2. 数据源分层

## 2.1 官方一手源（优先级 P0）

- Lucasfilm / StarWars.com 官方页面（电影、角色、时间线、公告）。
- 官方出版物目录（小说、设定集、漫画发行信息）。
- 官方社媒与发布会素材（预告、访谈、角色设定）。

## 2.2 权威二手源（优先级 P1）

- 出版社与发行方元数据（ISBN、出版时间、语言版本）。
- 主要电影数据库（用于上映时间、演员信息交叉校验）。

## 2.3 社区补充源（优先级 P2，需审核）

- 粉丝 Wiki 与论坛讨论，仅作线索，不直接入库。
- 入库前必须有 P0/P1 证据链。

## 3. 多语种采集流程

1. **英文主干采集**：优先采英文原始信息，建立 canonical 记录。
2. **中文对齐翻译**：中文字段由术语表 + 人工审校生成。
3. **术语库管理**：维护人名、地名、阵营名、头衔映射（en ↔ zh-CN）。
4. **证据追溯**：每个字段记录 `source_url`、`source_lang`、`retrieved_at`。

## 4. 数据治理与质量

- 字段级置信度：`high/medium/low`。
- 冲突检测：同一实体多来源冲突时标记待审核。
- 版本控制：每次更新形成快照，可回滚。
- 合规要求：遵循版权与引用规范，仅存储必要摘要与结构化事实。

## 5. 素材资产管理（图片/视频/海报）

- 资产只保存授权可用资源或引用链接，不存侵权原图。
- 元数据字段：`copyright_owner`, `license_type`, `usage_scope`, `expiry_date`。
- CDN 分发多尺寸缩略图，原图冷存储。

## 6. 持续更新机制（应对后续新作）

- 建立“新作监测任务”：按周扫描官方发布。
- 新作品触发自动建档：
  - 创建 `work` 实体
  - 关联预告、角色、事件占位节点
  - 发布后补完剧情与关系
- 对小说/漫画使用章节级导入模板，支持逐章扩展。

## 7. 团队分工建议

- 采集团队：信息抓取与初步结构化。
- 编辑团队：术语统一与事实校验。
- 审核团队：权威性与版权合规把关。
- 工程团队：采集管线、图谱同步与发布自动化。
