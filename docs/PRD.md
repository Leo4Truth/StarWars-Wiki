# 《星球大战》系列电影 Wiki 网站 PRD（产品需求文档）

## 1. 项目概述

- **项目名称**：StarWars Wiki（前后端分离）
- **目标用户**：
  - 星战新观众：希望快速理解人物关系与时间线。
  - 核心粉丝：希望深入检索人物、事件、阵营关系。
  - 内容创作者：希望获取结构化资料并进行可视化展示。
- **产品目标**：
  1. 提供覆盖《星球大战》系列电影（天行者传奇 + 关键外传）的结构化知识库。
  2. 提供互动式知识图谱，支持“人物-事件-阵营-地点-电影”多维探索。
  3. 为关键人物提供编年史式页面，按时间组织其生平与重大节点。

## 2. 产品范围

### 2.1 In Scope（首期）

1. 首页（搜索 + 入口导航 + 推荐人物/电影）。
2. 电影列表页与电影详情页。
3. 人物列表页与人物详情页（编年史时间线）。
4. 互动知识图谱页面（可筛选、可缩放、可点击联动）。
5. 基础后台管理能力（数据导入、修订、发布）。

### 2.2 Out of Scope（后续迭代）

- 用户登录/收藏体系。
- 社区评论、UGC 编辑协作。
- 多语言全文国际化（首期仅中文 UI，可预留字段）。

## 3. 用户故事

1. 作为新观众，我希望输入“卢克”后快速看到其出场电影、师徒关系和关键事件。
2. 作为粉丝，我希望在图谱中点击“达斯·维达”时，自动高亮其家族、阵营变化和对应电影。
3. 作为内容创作者，我希望按“银河帝国”筛选人物并导出时间线信息用于二次创作。

## 4. 核心功能需求

## 4.1 信息架构

- 一级导航：`首页` / `电影` / `人物` / `知识图谱` / `关于`
- 全局搜索：支持人物、电影、地点、阵营关键词。
- 面包屑导航：支持跨页面回溯（如：人物 > 阿纳金 > 事件节点）。

### 4.2 人物编年史页面（必须）

**页面模块**：
1. 人物头图与摘要（中英文名、阵营、种族、状态）。
2. 生平时间线（按 BBY/ABY 或电影内年份排序）。
3. 关系网络（家族、师徒、宿敌、盟友）。
4. 关键战役/事件卡片（关联电影、地点、结果）。
5. 媒体资料（剧照、台词节选、引用来源）。

**交互要求**：
- 时间线节点可展开详情。
- 节点点击后联动右侧“关系网络”与“相关电影”面板。
- 支持“仅显示电影正史（Canon）”。

### 4.3 知识图谱可视化页面（必须）

**实体类型**：
- 人物（Character）
- 电影（Film）
- 阵营（Faction）
- 地点（Planet/Location）
- 事件（Event）

**关系类型**：
- `appears_in`（人物->电影）
- `belongs_to`（人物->阵营）
- `happened_at`（事件->地点）
- `involves`（事件->人物）
- `related_to`（人物<->人物）

**交互要求**：
- 缩放、平移、框选、悬停提示。
- 关系类型筛选 + 电影时期筛选（前传/正传/后传）。
- 点击节点后展示详情抽屉，并可跳转详情页。
- 图谱与搜索联动（搜索词命中后自动定位高亮节点）。

### 4.4 搜索与过滤

- 支持拼音/中文别名（例如：Anakin / 阿纳金 / 达斯维达）。
- 支持组合过滤：阵营 + 电影 + 实体类型。
- 支持排序：热度、时间顺序、字母顺序。

### 4.5 管理后台（MVP）

- 数据导入：JSON/CSV 批量导入。
- 数据版本：草稿/已发布。
- 审计日志：记录字段变更与操作人。

## 5. 非功能需求

1. **性能**：图谱首屏加载 < 3s（典型数据量 5k 节点 / 20k 边以内采用分层加载）。
2. **可用性**：移动端适配（图谱页降级为简化模式）。
3. **可访问性**：关键交互可键盘操作，色彩对比符合 WCAG AA。
4. **SEO**：人物/电影详情页支持 SSR 或 SSG 预渲染策略。
5. **安全性**：管理接口鉴权、输入校验、防注入。

## 6. 数据与内容规划

### 6.1 关键实体字段（示例）

- Character:
  - id, name_zh, name_en, aliases, faction_ids, species, birth_time, death_time, bio
- Film:
  - id, title_zh, title_en, release_date, era, synopsis
- Event:
  - id, name, time_label, location_id, summary, film_id

### 6.2 内容优先级

- P0 人物（首批）：卢克、莱娅、汉·索罗、欧比旺、尤达、阿纳金/达斯·维达、帕尔帕廷、雷伊、凯洛·伦等。
- P1 地点：塔图因、霍斯、达戈巴、科洛桑、死星相关地点。
- P1 事件：克隆人战争关键节点、义军崛起、绝地归来关键战役。

## 7. 里程碑与验收

### 7.1 里程碑

1. **M1（第 1-2 周）**：信息架构 + 数据模型 + API 草案。
2. **M2（第 3-4 周）**：人物页与电影页 MVP 完成。
3. **M3（第 5-6 周）**：知识图谱交互完成并联动搜索。
4. **M4（第 7 周）**：性能优化、可用性修复、上线准备。

### 7.2 验收标准（核心）

- 可通过搜索在 2 步内到达任一关键人物详情页。
- 关键人物页面均含时间线，且节点不少于 5 个。
- 图谱页面支持至少 3 类筛选与点击跳转。
- 后台可成功导入至少 200 条结构化实体数据。

## 8. 多语言与持续扩展要求（新增）

### 8.1 多语言（首期 zh-CN + en-US）

- 首期必须支持简体中文与英文双语。
- 每个页面应具备稳定 URL 规则，例如：
  - `/zh-CN/characters/luke-skywalker`
  - `/en-US/characters/luke-skywalker`
- UI 视觉风格在不同语言版本保持一致，仅内容与排版细节按语言自适应。
- 搜索支持跨语言别名映射（如 Luke Skywalker ↔ 卢克·天行者）。

### 8.2 后续知识扩展能力

- 数据模型必须支持新增作品类型：电影、剧集、小说、漫画、游戏。
- 内容入库采用“作品 -> 章节/事件 -> 实体关系”可扩展结构。
- 每次新增官方作品时，可在不改动核心 schema 的前提下完成增量导入。

### 8.3 验收补充

- 任意核心页面可在中英文之间一键切换且路由有效。
- 新增一个“作品类型”不会导致现有图谱与时间线页面改版。
