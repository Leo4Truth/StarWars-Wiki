# 《星球大战》Wiki 可视化页面 UI/UX 原型设计

## 1. 设计目标

- 让用户“从关系理解故事”，而不仅是阅读条目。
- 通过视觉层级减少复杂世界观的理解成本。
- 支持桌面端深度探索与移动端快速浏览。

## 2. 设计系统（简版）

- **主色**：
  - 深空黑 `#0B0F1A`
  - 光剑蓝 `#3BA8FF`
  - 原力金 `#F5C451`
- **语义色**：
  - 绝地系（蓝绿）
  - 西斯系（红橙）
  - 中立系（灰紫）
- **字体**：中文无衬线（如 Noto Sans SC），英文标题可使用科幻风 Display 字体。
- **间距体系**：8px 网格。

## 3. 核心页面原型

## 3.1 首页（Home）

**布局结构（桌面）**：
1. 顶部导航栏：Logo + 一级导航 + 全局搜索。
2. Hero 区：主题语 + 快速入口按钮（人物、电影、图谱）。
3. 今日推荐：3 张人物卡 + 3 张电影卡。
4. 世界观入口：按时代分组（前传/正传/后传）。

**交互细节**：
- 搜索框输入时显示实体类型标签（人物/电影/地点）。
- 鼠标悬停人物卡展示“阵营 + 代表事件”。

## 3.2 人物编年史页（Character Timeline）

**页面框架（桌面三栏）**：
- 左栏（20%）：人物基础信息（头像、别名、阵营标签、统计信息）。
- 中栏（55%）：纵向时间线（Chronology Rail）。
- 右栏（25%）：关系网络迷你图 + 相关电影列表。

**时间线卡片结构**：
- 时间标签（BBY/ABY）
- 事件标题
- 摘要描述（2-3 行）
- 关联实体 Tag（电影/地点/人物）

**关键交互**：
- 点击时间节点：中栏展开详情；右栏联动高亮相关人物。
- “正史/扩展宇宙”切换开关。
- 吸顶小导航：快速跳转“童年/转折/终局”等章节。

## 3.3 知识图谱页（Knowledge Graph）

**页面框架**：
1. 顶部工具条：搜索框、时代筛选、关系筛选、布局切换。
2. 主画布区：力导向图谱（可缩放平移）。
3. 右侧详情抽屉：节点详情 + 快速跳转按钮。
4. 底部小地图：当前视口位置。

**视觉编码建议**：
- 节点形状区分实体：
  - 人物：圆形
  - 电影：矩形
  - 阵营：六边形
  - 地点：菱形
  - 事件：胶囊形
- 边样式区分关系类型（实线/虚线/箭头）。
- 节点大小映射“连接度”或“影响力”。

**图谱交互原型流程**：
1. 用户搜索“阿纳金”。
2. 画布自动定位并高亮该节点。
3. 第一圈邻接节点展开（欧比旺、帕尔帕廷、卢克、相关电影）。
4. 点击“达斯·维达”身份节点后显示“身份转变事件链”。
5. 点击抽屉中的“查看完整编年史”跳转人物页。

## 3.4 移动端适配策略

- 图谱页默认进入“聚焦模式”（单节点 + 邻居列表），减少性能消耗。
- 时间线改为折叠卡片流，支持“按时代筛选”。
- 底部 Tab 导航替代顶部复杂导航。

## 4. 可用性与无障碍

- 图谱节点 hover 信息在移动端改为点击弹窗。
- 颜色之外增加图标和边框样式区分阵营，避免纯色依赖。
- 键盘支持：
  - `Tab` 聚焦节点
  - `Enter` 打开节点详情
  - `Esc` 关闭详情抽屉

- 设计工具：Figma（组件化 + 自动布局）。
- 原型页面清单：

  2. Character Detail (Timeline)
  3. Knowledge Graph
  4. Film Detail
  5. Mobile Graph Focus Mode
- 输出规格：
  - Desktop: 1440px
  - Tablet: 1024px
  - Mobile: 390px

## 6. 多语言一致性设计（新增）

- 语言切换器位置固定在顶部导航右侧，采用 `中 / EN` 双态展示。
- 中英文版共用同一组件库与设计 Token（颜色、圆角、阴影、间距）。
- 文案长度差异通过自适应布局处理：
  - 按钮最小宽度 + 内容溢出省略。
  - 卡片模块支持自动增高。
- 字体策略：中文优先 Noto Sans SC，英文优先 Inter / Roboto，保持基线与字重一致。
